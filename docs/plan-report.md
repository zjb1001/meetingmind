# 📋 CEO 产品规划报告 - 智能会议纪要系统

## 项目代号: MeetingMind

---

## 一、需求深度分析

### 1.1 用户痛点

| 痛点 | 现状 | 影响 |
|------|------|------|
| 会议记录效率低 | 人工记录耗时耗力 | 会后整理时间 > 会议时间 |
| 遗漏关键信息 | 人耳处理速度有限 | 行动项遗漏率达30%+ |
| 多语言会议 | 跨国团队沟通障碍 | 理解偏差、执行错误 |
| 纪要格式混乱 | 无标准模板 | 难以追踪执行进度 |

### 1.2 核心需求拆解

```
输入层: Teams/耳机音频流 → 语音捕获
       ↓
处理层: 语音识别(ASR) → 语言检测 → 说话人分离 → 语义理解
       ↓
输出层: 结构化纪要(主体+动作+时间) → 任务分发 → 归档查询
```

### 1.3 用户价值评估

| 价值维度 | 评分 | 说明 |
|----------|------|------|
| 时间节省 | ⭐⭐⭐⭐⭐ | 90%会议纪要时间节省 |
| 执行效率 | ⭐⭐⭐⭐ | 行动项自动追踪，减少遗忘 |
| 跨国协作 | ⭐⭐⭐⭐⭐ | 实时翻译+多语言纪要 |
| 知识沉淀 | ⭐⭐⭐⭐ | 可搜索的历史会议库 |

**综合价值: 10星级产品** ✅

## 二、产品方案设计

### 2.1 MVP 方案 (2周)

**核心功能**:
- 本地音频录制（虚拟音频设备捕获）
- 集成智谱/Whisper API 语音识别
- 基础纪要模板（参会人+议题+行动项）
- Markdown 格式输出

**技术栈**:
- Python + PyAudio
- 智谱 GLM-4-Plus (ASR + 理解)
- 本地文件存储

**成本**: ¥0（使用现有Token）

### 2.2 标准版 (1个月)

**新增功能**:
- 实时转录（流式识别）
- 说话人分离（声纹识别）
- 多语言自动检测
- Teams 插件集成
- 任务自动同步（飞书/钉钉/邮件）

**技术栈**:
- 本地 ASR (Whisper.cpp)
- Speaker Diarization (pyannote)
- Teams Graph API
- SQLite + 本地向量库

**成本**: 开发成本 ¥5k，运行成本 ¥100/月

### 2.3 10星级方案 (3个月)

**创新功能**:
- 端到端加密（企业级安全）
- 智能摘要（关键决策点提取）
- 情绪分析（会议氛围评估）
- 自动任务追踪（与JIRA/飞书项目联动）
- 知识图谱（会议关联分析）
- 硬件级集成（会议麦/耳机固件）

**技术栈**:
- 私有化部署大模型
- 边缘计算（本地AI芯片）
- 企业SSO集成
- 分布式存储

**成本**: 开发成本 ¥50k+，运行成本 ¥2k/月

## 三、技术可行性分析

### 3.1 ASR 技术选型

| 方案 | 准确率 | 延迟 | 成本 | 隐私 |
|------|--------|------|------|------|
| Whisper API | 95% | 2-5s | ¥0.006/分钟 | 低 |
| 智谱 GLM-4 | 92% | 1-3s | 现有Token | 中 |
| Whisper.cpp (本地) | 90% | 实时 | 0 | **高** |
| 科大讯飞 | 93% | 1s | ¥0.008/分钟 | 低 |

**推荐**: MVP用智谱API，标准版用Whisper.cpp本地部署

### 3.2 说话人分离

| 方案 | 准确率 | 资源占用 | 说明 |
|------|--------|----------|------|
| pyannote.audio | 85% | 中 | 开源，支持实时 |
| Resemblyzer | 80% | 低 | 轻量级 |
| 云端API | 90% | 无 | 成本高 |

### 3.3 音频捕获方案

```
方案A: 虚拟音频设备 (VB-Cable/BlackHole)
        ├─ 优点: 通用性强，捕获系统音频
        └─ 缺点: 需要用户配置

方案B: Teams Bot (Graph API)
        ├─ 优点: 官方集成，自动获取
        └─ 缺点: 仅限Teams，需企业权限

方案C: 硬件钩子 (耳机固件)
        ├─ 优点: 质量最高，无配置
        └─ 缺点: 需要硬件支持

推荐: MVP用方案A，标准版支持方案B
```

## 四、模块设计

### 4.1 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                      MeetingMind System                      │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │ Audio Capture│  │  Real-time   │  │   Output     │      │
│  │   Layer      │→ │   Process    │→ │   Layer      │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
├─────────────────────────────────────────────────────────────┤
│  Capture Module │  ASR Engine   │  LLM Summarizer │ Export │
│  - PyAudio      │  - Whisper    │  - 主体提取     │ - MD   │
│  - Virtual Cable│  - 智谱API    │  - 行动项识别   │ - PDF  │
│  - Teams Bot    │  - 本地模型   │  - 时间解析     │ - API  │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 核心模块

| 模块 | 功能 | 技术方案 |
|------|------|----------|
| **Audio Capture** | 音频捕获 | PyAudio + 虚拟音频设备 |
| **VAD** | 语音活动检测 | webrtcvad / Silero |
| **ASR** | 语音识别 | Whisper.cpp / 智谱API |
| **Diarization** | 说话人分离 | pyannote.audio |
| **NLP** | 语义理解 | 智谱 GLM-4 |
| **Summarizer** | 纪要生成 | 结构化Prompt工程 |
| **Export** | 输出格式化 | Markdown / JSON |

### 4.3 数据流

```
Raw Audio (PCM)
    ↓
[VAD] → 分割有效语音段
    ↓
[ASR] → 转文本 (带时间戳)
    ↓
[Diarization] → 标记说话人
    ↓
[LLM] → 结构化理解
    ↓
输出: {
    "participants": ["张三", "李四"],
    "topics": [
        {
            "title": "项目进度",
            "summary": "...",
            "actions": [
                {"who": "张三", "what": "完成设计文档", "when": "2026-03-20"}
            ]
        }
    ]
}
```
